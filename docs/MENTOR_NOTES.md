# Mentor Notes

## Why this project exists
I built this as an end-to-end data workflow demo: validation, cleaning, reporting, and visual explanation.

## Technical decisions
- Used data contracts before visualization to avoid misleading charts.
- Separated ETL, algorithms, and visualization modules to keep responsibilities clear.
- Added algorithm visualizers to explain behavior, not just output.

## Build vs polish
- Build phase: cleaning pipeline, metrics, interactive charts, tests.
- Polish phase: updated screenshots, README clarity, stability fixes.

## What I learned
- Reliable analytics starts with input validation.
- Visual tools become stronger when backed by testable logic.
